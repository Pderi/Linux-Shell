# myshell 小组分工文档

> 项目：操作系统课程设计 · 实验一 Shell 编程  
> 小组人数：3 人  
> 原则：**代码量与功能点均尽量三等分**

---

## 分工总览

| 成员 | 代码行数（约） | 负责功能（4 组） |
|------|----------------|------------------|
| **秦健超** | ~420 行 | 命令解析、主控、读入、exit |
| **张孙毓** | ~430 行 | 外部执行、I/O、history、Tab 补全 |
| **郭沛特** | ~410 行 | cd、echo、type、alias |

---

## 成员与负责模块

### 秦健超 — 命令解析与主控

| 文件 | 职责 |
|------|------|
| `src/main.c` | 主循环、模块初始化与退出 |
| `src/parser.c`、`include/parser.h` | 解析命令行：引号、`\|`、`>`、`>>`、`<`、`&` |
| `src/shell.c`、`include/shell.h` | 内置 / 外部分发、`exit` 退出标志 |
| `src/input.c`、`include/input.h` | 提示符与 readline / getline 读入 |

**负责功能：**

1. **命令解析** — 引号、管道、重定向、后台
2. **Shell 主控** — 主循环、模块初始化、命令分发
3. **读入与提示符** — 交互 / 脚本双模式
4. **exit** — 退出 Shell

---

### 张孙毓 — 进程执行与交互 I/O

| 文件 | 职责 |
|------|------|
| `src/executor.c`、`include/executor.h` | fork / exec / wait、管道、重定向、后台、内置 fork 执行 |
| `src/history.c`、`include/history.h` | 历史记录存储、`history` 命令、持久化 |
| `src/completion.c`、`include/completion.h` | Tab 命令 / 路径补全 |

**负责功能：**

1. **外部命令** — `ls`、`cat`、`grep`（fork + execvp）
2. **I/O 机制** — `>` / `>>` / `<`、管道 `\|`、后台 `&`
3. **history** — 环形缓冲、`history n`、上下键、持久化
4. **Tab 补全** — 命令名 + 路径补全

---

### 郭沛特 — 内置命令

| 文件 | 职责 |
|------|------|
| `src/builtins.c`、`include/builtins.h` | 内置命令调度：`cd`、`echo`、`exit` |
| `src/type.c`、`include/type.h` | `type` 命令、内置表、PATH 查找 |
| `src/alias.c`、`include/alias.h` | 别名存储 / 展开、`alias` / `unalias` 命令 |

**负责功能：**

1. **cd** — 目录切换，支持 `~`
2. **echo** — 输出，支持 `$VAR` / `${VAR}`
3. **type** — 区分内置 / 外部命令
4. **alias / unalias** — 别名定义、展开、取消

## 共同事项

三人共同完成：整体联调、`make test` 功能测试、课程报告、系统演示。

| 事项 | 建议分工 |
|------|----------|
| 课程报告 | 各写约 1/3：秦=设计+解析，张=进程+I/O+history+补全，郭=内置命令 |
| 系统演示 | 各 2–3 分钟（见 `docs/项目讲解文档.md` 第六节） |
| 自动化测试 | 张孙毓维护 `scripts/run_tests.sh` 中与 executor/history 相关用例 |

---

## 协作说明

- 公共数据结构 `Command`、`Pipeline` 定义在 `parser.h`，由**秦健超**维护
- `run_builtin()` 在 `builtins.c`（郭沛特）中统一调度，各模块提供 `builtin_xxx()` 接口
- 修改 `include/*.h` 中与他人相关的接口时，提前在组内同步
- 联调顺序：秦健超（解析 + 主循环）→ 郭沛特（内置命令）→ 张孙毓（外部执行 + I/O）
