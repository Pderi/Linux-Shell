## 实验一：Shell 编程

### 实验目的

学习如何编写一个 Linux 的外壳——Shell，即命令解释程序。学习怎样接受命令、解释命令、执行命令，特别是采用**创建子进程**的方式来执行一个程序，以及**父进程如何继续子进程的工作**。

### 任务要求

- 理解和掌握 Linux 中 shell 的原理和实现；
- 编写一个 **C 语言程序**作为 Linux 内核的 shell 命令行解释程序；
- 实现 Shell 的 **ls、cd、cat、grep、echo、type、history、alias、命令补全、后台运行、管道与重定向**的功能。

### Shell 初识

Shell 是一种具备特殊功能的程序，它是介于使用者和 UNIX/Linux 操作系统之核心程序（kernel）间的一个接口。

为了对用户屏蔽内核的复杂性，也为了保护内核以免用户误操作造成损害，在内核的周围建了一个外壳（shell）。用户向 shell 提出请求，shell 解释并将请求传给内核。

#### Shell 提示符

| 符号 | 含义 |
|------|------|
| `$` | 普通用户 |
| `#` | 超级用户 |

提示符通常显示：用户名 @ 系统名 当前目录名

退出方式：`exit` 或者 `Ctrl+D`

#### Shell 种类

- **Bash**（以及早期的 sh）：来自于 AT&T 贝尔实验室 Bourne 创建的 sh，在 GNU 项目的资助下 Brian Fox 开发了 bash，之后 Chet Ramey 使其得到发展。
- **tcsh**（以及早期的 csh）：是 C Shell（csh）的开放源代码版本，采用的语法类似于 C 编程语言，被大多数 Berkeley UNIX 系统作为默认 Shell。
- **Korn shell（ksh）**：AT&T 贝尔实验室的 David Korn 开发，它集合了 C shell 和 Bourne shell 的优点，并且和 Bourne shell 完全兼容，是 UNIX System V 系统默认的 Shell。

在 `/etc/shells` 文件中，列出了目前系统可以使用的 shell，并且给出了这些 shell 程序的位置。

### Shell 编程的基本功能

#### (1) Shell 命令解释及执行

用户登录成功后，实际进入 Shell，它遵循一定的语法将输入的命令解释并传递给系统。

**命令格式：**

```bash
命令名 [选项] [参数]
```

- **选项**：更改命令的行为，形式为 `-字母`；例如：`ls -l`
- **参数**：命令使用的一条额外信息；例如：`cat /etc/passwd`
- **环境变量**：对用户 Shell 可能有用的信息；例如：`echo $HOME`
- **元字符**：对 Shell 有特殊意义的字符；例如：`cat /etc/passwd | sort`

#### (2) 命令补全功能

所谓「命令补全」是指在用户输入命令的时候，无需输入完整的命令行，Linux 系统的 shell 会自动查找出最符合的命令名称，供用户选择。这样的功能可以帮用户节省输入长串命令的时间。

**示例：**

在 `/root` 目录下有一个 `testexecvp.c` 文件，输入：

```bash
more /root/teste<Tab>
```

bash 会自动补足为：

```bash
more /root/testexecvp.c
```

更多示例：

```bash
echo Fi<Tab>    # → echo File
cd ~ro<Tab>     # → cd root/  补全根用户的主目录
echo $OS<Tab>   # → echo $OSTYPE
```

如果系统中有多个文件都与输入的前缀相同，当用户**连续按下 2 次 Tab 键**时，系统会显示当前目录下所有具有相同前缀的文件名称，供用户选择。

例如，输入 `more pa` 后按 2 次 Tab 键，会列出所有匹配的文件名。

#### (3) 查阅历史记录

- `history` 命令：每当用户输入的命令并按下 Enter 后，都会被记录在命令记录表中。
- 默认情况下，bash 使用的命令记录表文件为用户主目录下的 `.bash_history`（文件名前面的 `.` 表示这是一个隐藏文件）。
- 可以使用环境变量 `HISTSIZE` 来定义命令记录表的条数，默认的记录条数为 **1000** 条。

可以直接浏览 `.bash_history` 文件，或使用 `history` 命令来查看目前的命令记录。

使用 `history n` 命令，其中 n 表示需要查看的最近执行的命令的条数。例如：列出系统最近执行的 10 条命令。

**运行历史命令（扩展）：**

| 语法 | 含义 |
|------|------|
| `!n` | 运行第 n 条历史命令 |
| `!!` | 运行前一条指令 |
| `!?String?` | 运行包含 string 的命令 |

#### (4) 别名功能

别名功能是指提供给用户使用自定义的简单字符串，来替换复杂的命令选项，或是替换连续多个命令的连续组合的功能，从而使得用户可以自定义符合自己习惯的操作命令。

**格式：**

```bash
alias 别名='命令'
```

**创建别名示例：**

如果希望使用 `dir` 来代替 `ls -l`：

```bash
alias dir='ls -l'
```

**查看别名：** 直接输入 `alias` 命令。

**取消别名：**

```bash
unalias dir
```

#### (5) 后台处理

Linux 是多用户多任务的操作系统，它允许多个用户同时登录系统，也允许多个程序同时执行。但因为 shell 使用交互式模式，目前执行的命令会一直掌握系统的控制权，直到该程序结束为止，这类程序称为**前台程序**。

只要在输入命令时，在命令后面加上 **`&`** 符号，系统就会开始以后台的方式执行该命令，屏幕将显示该后台运行程序的进程 PID，然后 shell 将回到命令提示符状态，以等待用户的下一个命令的输入。

**格式：**

```bash
命令 &
```

**示例：** 将 `ls -l` 命令后台处理，屏幕显示后台进程 PID。

#### (6) I/O 重定向

标准输入和输出有 3 种形态：

| 流 | 说明 |
|----|------|
| 标准输入（stdin） | 通常是指键盘 |
| 标准输出（stdout） | 通常是指将命令执行的结果输出到终端机或屏幕上 |
| 标准错误输出（stderr） | 是指在命令发生错误时，将其错误信息输出到屏幕上 |

**输出重定向：**

```bash
命令 > 文件
```

例如：将 `ls -l` 命令的结果送至 File1 文件中。

**输入重定向：**

```bash
命令 < 文件
```

例如：`cat` 从 File1 获得输入，察看 File1 文件的内容。

## 实验二：Linux 二级文件系统

### 实验目的

1. 通过一个简单多用户文件系统的设计，加深理解文件系统的内部功能和内部实现。
2. 结合数据结构、程序设计、计算机原理等课程的知识，设计一个多级目录文件系统，进一步理解操作系统。
3. 通过对实际问题的分析、设计、编程实现，提高学生实际应用、编程的能力。

### 任务要求（模仿 Linux 的 EXT2 文件系统）

- 理解和掌握 EXT2 文件系统的原理和实现；
- 创建模拟文件系统（使用二进制文件模拟文件系统），模拟磁盘空间，所有用户信息、inode 信息、超级块信息、文件信息；
- 实现文件和目录操作：**dir、create、delete、open、close、read、write** 等命令，其中 dir 列目录时要列出**文件名、物理地址、保护码和文件长度**；
- 支持用户交互，包括：**login** 用户登录；
- **安全性**：源文件可以进行读写保护（读写权限）。

### Linux 文件系统的设计思路

| 结构 | 说明 |
|------|------|
| Superblock（超级块） | 存放文件系统本身的结构信息，例如 block 和 inode 的总量，未使用的 block 和 inode 的数量，block 和 inode 的大小等 |
| Group Description Table（组描述符） | 用来描述每个组的开始与结束位置的块号码，以及说明每个块分别介于哪一个 block 号码之间 |
| Block bitmap（块位图） | 记录哪个数据块已经被占用，哪个数据块没有被占用 |
| Inode bitmap（inode 位图） | 记录的是使用与未使用的 inode 号 |
| Inode table（inode 表） | 存放着所有文件的 inode 信息 |
| Data Blocks（数据块） | 来存放文件内容 |

#### Inode 节点中存储信息

- 文件的读写权限（rwx）
- 文件的拥有者和所属组（owner/group）
- 文件的容量
- 文件的 ctime（创建时间）
- 文件的 atime（最近一次的读取时间）
- 文件的 mtime（最近修改的时间）
- 文件真正内容的指向（12 个直接 + 1 个间接 + 1 个双间接 + 1 个三间接记录区）

## 实验三：Linux 系统调用

### 实验目的

学习系统调用的执行流程，理解 Linux 系统调用。

### 任务要求

设计并实现进程全量监控与分析系统，要求设计并实现 **3 个自定义系统调用**和 **1 个用户态监控程序**：

| 系统调用 | 功能 |
|----------|------|
| `sys_proc_collect` | 一次性收集所有进程的完整信息 |
| `sys_proc_snapshot` | 基于进程树，返回父子关系拓扑图 |
| `sys_proc_stat` | 返回系统整体进程统计 |

**用户态监控程序：**

- 必须通过 `syscall()` 调用上述 3 个自定义系统调用；
- **禁止**调用 ps、`/proc` 文件读取、eBPF、ptrace 等旁路采集手段。

**具体功能包括：**

- 多条件过滤显示（按 PID、进程名、状态）
- 实时刷新（每秒刷新一次）进程状态快照
- 进程排序功能（按 CPU 时间 / 内存使用量排序）
- 进程树导出功能（显示带缩进的父子进程树，导出至本地文本文件）

## 实验四：Linux 驱动程序

### 实验目的

理解 Linux 内核模块化机制、设备驱动注册原理、内核态与用户态数据交互方式，掌握无重启内核的内核功能拓展技术。

### 任务要求

开发一个**虚拟字符设备驱动程序**，实现设备的 open、read、write、close 内核回调函数。

> 备注：实验也可以实现其他任意设备的驱动程序。

**包括设备号管理、字符设备注册、自动创建设备文件等模块，要求：**

- 用户态可向虚拟设备写入数据，内核态接收数据并处理，可读取内核返回的设备状态信息；
- 驱动内置一页大小内核缓冲区，用户 read 读缓冲区、write 写入缓冲区；
- 支持多进程打开同一设备，独立读写偏移；
- 支持自动分配设备号、自动创建设备文件；
- 支持模块动态加载、卸载、设备节点注册测试。
