# myshell 项目讲解文档

> 操作系统课程设计 · 实验一：Shell 编程  
> 本文档按三人分工讲解各模块的设计与代码实现。

| 成员 | 负责模块 |
|------|----------|
| **秦健超** | 命令解析、主控、读入（parser / shell / main / input） |
| **张孙毓** | 进程执行、I/O、history、Tab 补全（executor / history / completion） |
| **郭沛特** | 内置命令（builtins / type / alias） |

---

## 一、项目总览（共同部分）

### 1.1 Shell 是什么

Shell 是用户与 Linux 内核之间的**命令解释器**，负责读入命令、解析命令、执行命令。

```
用户输入一行命令
    ↓
myshell 读入 → 解析 → 分发 → 执行
    ↓                    ├─ 内置命令：Shell 进程内直接运行
    ↓                    └─ 外部命令：fork 子进程 + exec 运行 ls 等
```

### 1.2 三人协作的整体流程

```
┌─ 秦健超：input_read_line() 读入一行 ─────────────────────────┐
│  郭沛特：alias_expand_line() 展开别名                         │
│  秦健超：parse_line() 解析为 Pipeline（失败则跳过，不记历史）   │
│  张孙毓：history_add() 记录原始输入（解析成功后）              │
│  秦健超：shell_execute() 分发                                 │
│      ├─ 内置 → 郭沛特：run_builtin()                          │
│      └─ 外部/管道 → 张孙毓：execute_pipeline()                 │
└──────────────────────────────────────────────────────────────┘
```

### 1.3 公共数据结构（秦健超定义，三人共用）

定义在 `include/parser.h`：

```c
typedef struct {
    char *argv[MAX_ARGS];   /* 参数数组，argv[0] 为命令名 */
    int   argc;
    char *in_file;          /* 输入重定向：< file */
    char *out_file;         /* 输出重定向：> file 或 >> file */
    int   append;           /* 1=追加(>>)，0=覆盖(>) */
} Command;

typedef struct {
    Command cmds[MAX_PIPELINE];  /* 管道各段命令，最多 16 段 */
    int     count;               /* 管道段数 */
    int     background;          /* 1=行末有 & */
} Pipeline;
```

**示例：** `grep hello a.txt | wc -l > out.txt &` 解析为：

```
count = 2, background = 1
cmds[0] = { argv: ["grep","hello","a.txt"] }
cmds[1] = { argv: ["wc","-l"], out_file: "out.txt" }
```

## 二、秦健超 — 命令解析与主控

> 负责文件：`main.c`、`parser.c`、`shell.c`、`input.c` 及对应头文件

### 2.1 负责的功能

| 功能 | 说明 |
|------|------|
| 命令解析 | 拆分命令名、参数、`\|`、`>`、`>>`、`<`、`&` |
| 主循环 | 启动初始化、循环读命令、退出清理 |
| 命令分发 | 判断走内置还是外部执行路径 |
| 读入与提示符 | readline 交互 / getline 脚本模式 |
| `exit` | 设置退出标志，终止主循环 |

### 2.2 main.c — 主循环

**初始化（启动时执行一次）：**

```c
history_init(&g_history);          // 张孙毓模块
history_load(&g_history);
history_sync_readline(&g_history);
alias_init();                      // 郭沛特模块
completion_init();                 // 张孙毓模块
signal(SIGCHLD, SIG_IGN);          // 防止后台子进程变僵尸
```

**主循环（每条命令重复）：**

```c
for (;;) {
    raw = input_read_line();              // 秦健超：读入
    expanded = alias_expand_line(raw);    // 郭沛特：展开别名
    if (parse_line(expanded, &pl) < 0)    // 秦健超：解析（失败则跳过）
        continue;
    history_add(&g_history, raw);         // 张孙毓：记原始输入（非展开后）
    shell_execute(&pl);                   // 秦健超：分发执行
    parser_free_pipeline(&pl);            // 秦健超：释放内存
}
```

**退出时：** 调用张孙毓模块的 `history_save()`、`history_free()`，以及郭沛特模块的 `alias_free_all()`。

### 2.3 parser.c — 命令解析

**入口函数：** `parse_line(const char *line, Pipeline *pl)`

**解析步骤：**

```
原始字符串
  ① 去掉行末 &  →  background = 1
  ② 按 | 分段（引号内的 | 不拆分）
  ③ 每段调用 parse_segment() 解析参数和重定向
  ④ 填入 Pipeline 结构
```

**parse_segment() 识别的符号：**

| 符号 | 处理 |
|------|------|
| `<` | 下一个 token → `in_file` |
| `>` | 下一个 token → `out_file`（覆盖） |
| `>>` | 下一个 token → `out_file`（追加） |
| `"..."` / `'...'` | 引号内整体作为一个参数 |
| 普通文本 | 按空格拆成 argv[] |

**关键设计：** 管道分段时跳过引号内的 `|`，避免 `"a|b"` 被错误拆分（`parser.c` 约第 182–200 行）。

### 2.4 shell.c — 命令分发

**入口函数：** `shell_execute(Pipeline *pl)`

```c
int shell_execute(Pipeline *pl) {
    if (pl->count == 1 && is_builtin(pl->cmds[0].argv[0]))
        return executor_run_single_builtin(...);  // 张孙毓模块

    return execute_pipeline(pl);                // 张孙毓模块
}
```

**`exit` 与内置 + 重定向/后台：** 由张孙毓的 `executor_run_single_builtin()` 处理——`exit` 直接设置退出标志；带重定向或 `&` 的内置命令会 fork 子进程执行。

### 2.5 秦健超 — 功能与代码对照

| 功能 | 关键函数 | 文件 |
|------|----------|------|
| 命令解析 | `parse_line()` | `parser.c` |
| 引号 / 管道 / 重定向 / & 识别 | `parse_segment()` | `parser.c` |
| 主循环 | `main()` | `main.c` |
| 读入 / 提示符 | `input_read_line()` | `input.c` |
| 内置/外部分流 | `shell_execute()` | `shell.c` |
| exit 标志 | `shell_request_exit()` | `shell.c` |
| 数据结构 | `Command`、`Pipeline` | `parser.h` |

### 2.6 input.c — 读入与提示符

**函数：** `input_read_line()` — 交互模式使用 GNU readline；非 TTY（脚本 / `make test`）自动改用 `getline`。

**提示符格式：** `myshell:当前目录$`（root 用户为 `#`）

## 三、张孙毓 — 进程执行与交互 I/O

> 负责文件：`executor.c`、`history.c`、`completion.c` 及对应头文件

### 3.1 负责的功能

| 功能 | 说明 |
|------|------|
| 外部命令 | `ls`、`cat`、`grep` 等，通过 fork + execvp 执行 |
| 重定向 | `>`、`>>`、`<` |
| 管道 | `cmd1 \| cmd2 \| cmd3` |
| 后台运行 | 命令末尾 `&`，打印 PID 后不阻塞 |
| history | 环形缓冲、`history n`、持久化、上下键 |
| Tab 补全 | 命令名 + 路径补全 |

### 3.2 单条外部命令

**函数：** `execute_single()`（`executor.c`）

```
fork()
  ├─ 子进程：设置重定向 → execvp("ls", argv)  变成 ls 程序
  └─ 父进程：
       ├─ 前台 → waitpid() 等待子进程结束
       └─ 后台 → 打印 [pid] running in background，立即回到提示符
```

**fork / exec / wait 时序（以 `ls -l` 为例）：**

```
myshell（父进程 PID=1000）
  │
  ├─ fork() → 子进程 PID=1001
  │              └─ execvp("ls") → 子进程变为 /usr/bin/ls
  │              └─ ls 执行完毕，退出
  │
  ├─ waitpid(1001)  等待结束
  └─ 回到提示符
```

### 3.3 I/O 重定向

**函数：** `executor_setup_redirection()`（`executor.c` 第 12–36 行）

在子进程中、`execvp` **之前**调用：

| 语法 | 实现 |
|------|------|
| `cmd < file` | `open(file)` → `dup2(fd, STDIN_FILENO)` |
| `cmd > file` | `open(file, O_TRUNC)` → `dup2(fd, STDOUT_FILENO)` |
| `cmd >> file` | `open(file, O_APPEND)` → `dup2(fd, STDOUT_FILENO)` |

**原理：** `dup2` 把文件 fd 复制到标准输入(0)或标准输出(1)，之后 exec 的程序就从文件读写，而非终端。

### 3.4 管道

**函数：** `run_pipeline_children()`（`executor.c`）

以 `cmd1 | cmd2 | cmd3` 为例：

```
         pipe[0]          pipe[1]
cmd1 ──────────→ cmd2 ──────────→ cmd3
```

**实现步骤：**

1. 创建 `n-1` 条管道：`pipe(pipes[i])`
2. 为每段命令 `fork()` 一个子进程
3. 子进程内用 `dup2` 连接管道：
   - 非首段：stdin ← 前一条管道的读端
   - 非末段：stdout → 当前管道的写端
4. 关闭所有多余的 pipe fd
5. 设置本段重定向，然后 `execvp` 或 `run_builtin`（管道中的 echo 等内置命令）
6. 父进程 `waitpid` 等待全部子进程

### 3.5 后台运行

**解析：** 秦健超的 `parser.c` 识别行末 `&`，设置 `background = 1`。

**执行：**

- **单条命令：** `execute_single()` 中 fork 后不 wait，打印 PID
- **管道命令：** 额外 fork 一层，整个管道在后台子进程中运行，Shell 立即返回提示符

### 3.6 子进程统一入口

**函数：** `run_command_child()`（`executor.c`）

```c
static void run_command_child(Command *cmd) {
    executor_setup_redirection(cmd);       // 先重定向
    if (is_builtin(cmd->argv[0]))
        _exit(run_builtin(cmd));           // 管道中的内置命令
    execvp(cmd->argv[0], cmd->argv);       // 外部命令
    _exit(127);                            // exec 失败
}
```

### 3.7 history.c — 历史记录

**存储：** 环形缓冲区，最多 1000 条（可通过 `HISTSIZE` 环境变量调小）。

| 函数 | 作用 |
|------|------|
| `history_add()` | 解析成功后追加原始输入；同步到 readline |
| `builtin_history()` | `history` / `history n` 命令入口 |
| `history_load()` / `history_save()` | 启动加载 / 退出保存 `~/.myshell_history` |

### 3.8 completion.c — Tab 补全

| 光标位置 | 补全内容 |
|----------|----------|
| 行首（第一个词） | 内置命令名 + PATH 中可执行文件名 |
| 其他位置 | 文件/目录路径（readline 内置文件名补全） |

双 Tab 列出所有候选是 readline 默认行为。

### 3.9 张孙毓 — 功能与代码对照

| 功能 | 关键函数 | 使用的系统调用 / 库 |
|------|----------|---------------------|
| 外部命令 ls/cat/grep | `execute_single()` | fork, execvp, waitpid |
| 重定向 > >> < | `executor_setup_redirection()` | open, dup2 |
| 管道 \| | `run_pipeline_children()` | pipe, dup2, fork |
| 后台 & | `execute_single()` / 后台 fork | fork（不 wait） |
| 内置 + 重定向/后台 | `executor_run_single_builtin()` | fork, dup2, waitpid |
| history | `history_add()` / `builtin_history()` | readline history |
| Tab 补全 | `myshell_completion()` | readline |

## 四、郭沛特 — 内置命令

> 负责文件：`builtins.c`、`type.c`、`alias.c` 及对应头文件

### 4.1 负责的功能

| 功能 | 说明 |
|------|------|
| `cd` | 切换目录（内置，chdir） |
| `echo` | 输出文本，支持 `$HOME` 变量展开 |
| `type` | 判断命令是内置还是外部 |
| `alias` / `unalias` | 别名定义、展开、取消 |

### 4.2 builtins.c — cd / echo 与调度

**入口函数：** `run_builtin(Command *cmd)` — 统一调度各内置命令，再分发给对应模块。

| 命令 | 实现位置 | 关键实现 |
|------|----------|----------|
| `cd` | `builtins.c` | `chdir()`；支持 `~` 展开 |
| `echo` | `builtins.c` | `expand_env_print()` 解析 `$VAR` |
| `type` | `type.c` | `builtin_type()` |
| `history` | `history.c` | `builtin_history()` |
| `alias` / `unalias` | `alias.c` | `builtin_alias()` / `builtin_unalias()` |
| `exit` | `shell.c` | `shell_request_exit()` |

**为什么 cd 必须是内置命令：** `chdir()` 只改变调用进程的目录，fork 子进程再 cd 不会改变 Shell 自身工作目录。

### 4.3 type.c — type 命令

| 函数 | 作用 |
|------|------|
| `is_builtin()` | 判断是否为内置命令 |
| `builtin_names()` | 返回内置命令表，供 Tab 补全使用 |
| `find_external_path()` | 遍历 PATH 查找可执行文件 |
| `builtin_type()` | 输出 `xxx is a shell builtin` 或 `xxx is /path` |

### 4.4 alias.c — 别名

**存储：** 链表，节点为 `name → value`。

**展开时机：** 在 `main.c` 中、**解析之前**调用 `alias_expand_line()`，只替换行首第一个 token。`alias` / `unalias` 命令本身不做展开。

### 4.5 郭沛特 — 功能与代码对照

| 功能 | 关键函数 | 文件 |
|------|----------|------|
| cd | `builtin_cd()` | `builtins.c` |
| echo / $VAR | `builtin_echo()` | `builtins.c` |
| type | `builtin_type()` | `type.c` |
| alias / unalias | `builtin_alias()` / `builtin_unalias()` | `alias.c` |
| 别名展开 | `alias_expand_line()` | `alias.c` |
