## 1. 背景与目标

### 1.1 背景

当前 myshell 对 `ls`、`cat`、`grep` 通过 `execvp` 调用系统命令实现，不符合课程/老师要求。老师明确：

- **系统调用（syscall）**：允许且必须使用（`fork`、`open`、`read`、`chdir` 等）
- **系统命令（外部程序）**：禁止（`execvp("ls")`、`system("cat")` 等）

### 1.2 重构目标

| 编号 | 目标 | 验收标准 |
|------|------|----------|
| G1 | 删除所有 `execvp` 调用 | 源码中无 `execvp`/`system`/`popen` |
| G2 | 新增内建 `ls` / `cat` / `grep` / `pwd` | 不依赖 `/bin/*`，用系统调用实现 |
| G3 | 保留 Shell 核心机制 | 管道、重定向、后台、信号、history、alias 仍可用 |
| G4 | 修复已知 Bug | 见第 6 节 |
| G5 | 测试与文档同步 | `make test` 全过，文档描述与实现一致 |

### 1.3 非目标（不在本次范围）

- 实现 `!n` / `!!` 历史重跑（课程扩展）
- 实现 `2>` stderr 重定向
- 实现 `wc`、`sort` 等额外内建
- 替换 GNU Readline（允许继续使用）

## 3. 目标架构

### 3.1 执行流程

```
用户输入
  ↓
alias_expand_line()          # 别名展开
  ↓
parse_line()                 # 解析为 Pipeline
  ↓
shell_execute()
  ↓
所有命令段均为内建？
  ├─ 单内建、无重定向、非后台 → 父进程直接 run_builtin()
  ├─ 单内建、有重定向/后台   → 父进程 dup2 或 fork（见 3.3）
  ├─ 管道（≥2 段）           → fork 每段 + pipe + dup2 + run_builtin()
  └─ 非内建命令              → 报错 "command not found"（不 exec）
```

### 3.2 内建命令全景

| 命令 | 实现文件 | 核心系统调用 |
|------|----------|--------------|
| cd | `builtins.c` | `chdir`, `getenv` |
| echo | `builtins.c` | `getenv`, `write`(stdio) |
| pwd | `builtins.c`（新增） | `getcwd` |
| ls | `builtin_ls.c`（新增） | `opendir`, `readdir`, `stat`, `closedir` |
| cat | `builtin_cat.c`（新增） | `open`, `read`, `write`, `close` |
| grep | `builtin_grep.c`（新增） | `fopen`/`open`, `getline`, 字符串匹配 |
| type | `type.c` | 查内建表 |
| history | `history.c` | 内存 + `fopen` |
| alias / unalias | `alias.c` | 内存链表 |
| exit | `builtins.c` | 标志位 |

### 3.3 关键设计决策

#### 决策 1：禁止 exec，fork 保留

管道和后台仍需要 `fork` 创建子进程；子进程中调用 `run_builtin()`，不再 `execvp`。

#### 决策 2：前台内建 + 重定向不 fork

```c
// 父进程中：save fd → dup2 重定向 → run_builtin() → restore fd
```

避免 B1（waitpid 与 SIGCHLD 竞态），性能更好。

#### 决策 3：管道中的内建仍 fork

每段命令独立进程，通过 `pipe` + `dup2` 连接 stdin/stdout。

#### 决策 4：内建实现最小可用集

| 命令 | MVP 范围 | 可选增强 |
|------|----------|----------|
| cat | 无参读 stdin；有参读文件；多文件 | — |
| ls | 无参/`ls path`；支持 `-l` | `-a` |
| grep | `grep pat file`；无 file 读 stdin；子串匹配 | `-i` 忽略大小写 |
| pwd | 输出当前目录 | — |

#### 决策 5：type 与 Tab 不再识别外部命令

- `type wc` → `wc: not found`
- Tab 命令名只补内建表中的 11 个命令

## 5. 各模块详细设计

### 5.1 builtin_cat

```c
int builtin_cat(Command *cmd);
```

| 场景 | 行为 |
|------|------|
| `cat` | `read(stdin)` 循环写 stdout |
| `cat file` | `open` → read/write 循环 → `close` |
| `cat f1 f2` | 依次处理，任一失败 perror 并置 rc=1 |
| 配合 `<` / `\|` | executor 已 dup2，cat 只管 fd 0/1 |

### 5.2 builtin_ls

```c
int builtin_ls(Command *cmd);
```

| 场景 | 行为 |
|------|------|
| `ls` | 列出当前目录 |
| `ls /path` | 列出指定目录 |
| `ls -l` | 每项 `stat` 输出权限/大小/时间 |
| `ls /bin/ls`（单文件） | `stat` 判断非目录，直接输出路径 |

选项解析：扫描 `argv` 中以 `-` 开头的参数。

### 5.3 builtin_grep

```c
int builtin_grep(Command *cmd);
```

| 场景 | 行为 |
|------|------|
| `grep pattern file` | 逐行读 file，含 pattern 则输出 |
| `grep pattern` | 从 stdin 读（管道） |
| 多文件 | 输出前加 `file:` 前缀 |
| 无匹配 | 返回 1（与 grep 惯例一致） |

### 5.4 builtin_pwd

```c
int builtin_pwd(Command *cmd);
```

- `getcwd(buf, size)` → `puts(buf)`
- 失败 `perror("pwd")` 返回 1

### 5.5 executor 改造

**删除：**

```c
execvp(cmd->argv[0], cmd->argv);  // run_command_child 中
```

**改为：**

```c
if (is_builtin(cmd->argv[0])) {
    int rc = run_builtin(cmd);
    _exit(rc);
}
fprintf(stderr, "myshell: %s: command not found\n", cmd->argv[0]);
_exit(127);
```

**新增 `run_builtin_in_parent`：**

用于 `executor_run_single_builtin` 中前台 + 有重定向的场景：

1. `saved_in = dup(STDIN)` / `saved_out = dup(STDOUT)`
2. `executor_setup_redirection(cmd)`
3. `run_builtin(cmd)`
4. restore fd

**管道入口检查：**

```c
for (int i = 0; i < pl->count; i++)
    if (!is_builtin(pl->cmds[i].argv[0])) { ... not found ... }
```

### 5.6 type 改造

内建表更新为 11 个命令。`builtin_type` 逻辑简化为：

```
is_builtin(name) ? "xxx is a shell builtin" : "xxx: not found"
```

删除 `find_external_path()` 函数及 `include/type.h` 中的声明。

### 5.7 completion 改造

- `build_command_matches` 只遍历 `builtin_names()`，删除 PATH 扫描
- 删除 `rl_attempted_completion_over = 1`（命令补全路径）
- 文件名补全仍用 `rl_filename_completion_function`

### 5.8 alias 修复

`builtin_alias` 中：若单个 argv 解析失败且后续还有 argv，则逐段合并后再解析：

```
argv[1]="dir='ls" + argv[2]="-l'" → "dir='ls -l'" → 解析成功
```

## 6. Bug 修复方案

### B1：waitpid: No child processes

| 方案 | 说明 |
|------|------|
| **主方案** | 前台内建+重定向改用 `run_builtin_in_parent`，不 fork |
| **兜底** | `wait_pid_blocking` 中 `errno == ECHILD` 视为已回收 |

### B2：Tab 补全

- 去掉 `rl_attempted_completion_over = 1`
- 命令补全仅内建表，减少干扰项

### B3：alias 引号空格

- alias 模块 argv 合并（见 5.8）
- 可选：parser 层改进（非必须，合并方案更稳）

### B4：测试脚本

| 原用例 | 改为 |
|--------|------|
| `grep … \| wc -l` | `grep root /etc/passwd \| grep root` |
| `sleep 0.1 &` | `grep root /etc/passwd &` 或 `ls &` |
| `type ls` 断言 `"ls is"` | `"ls is a shell builtin"` |
| 新增 | `wc -l /etc/passwd` → `command not found` |
