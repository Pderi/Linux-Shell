# 实验一：Shell 编程 — 设计方案

> **版本变更（2026-07-09，相较 exec 外部命令版）**
>
> - `ls` / `cat` / `grep` 由「`execvp` 调系统命令」改为**内建 + 系统调用**（`builtin_*.c`）

> - 执行模块删除 `execvp`；非内建命令直接报 `command not found`
> - 架构图、模块设计、Tab 补全、开发/测试计划同步更新
> - 明确约束：**系统调用可以，系统命令不行**

| 实验要求 | 本方案实现方式 | 对应模块 |

## 2. 设计目标

### 2.1 功能目标

实现一个名为 `myshell` 的简易 Shell，具备：

1. 交互式命令行（提示符、读入、解析、执行）
2. 全部实验命令以内建 + 系统调用方式实现（不 exec 外部程序）
3. 单条及多条管道、输入/输出重定向
4. 后台任务启动
5. 历史记录与别名管理
6. Tab 命令/路径补全

### 2.2 学习目标（对应实验描述）

- 理解 Shell 作为用户与内核之间的接口
- 掌握 `fork` / `wait` 进程控制流程（管道与后台通过 fork 跑内建）
- 理解标准 I/O 重定向与管道的文件描述符机制
- 理解内建命令如何用系统调用直接访问内核服务（读目录、读文件等）

### 4.1 主流程

```
启动 myshell
  → 初始化 history、alias、readline
  → loop:
       显示提示符（如 myshell:/home/user$ ）
       readline 读入一行
       空行 → 继续
       写入 history
       alias 展开
       parser 解析 → Command 或 Pipeline
       若为内建命令 → run_builtin() 或 executor 中 fork 跑内建
       若非内建 → 报错 command not found
       直到 exit 或 Ctrl+D
  → 清理资源，退出
```

## 5. 数据结构设计

### 5.1 单条命令

```c
typedef struct {
    char **argv;        /* 参数数组，argv[0] 为命令名，以 NULL 结尾 */
    char *in_file;      /* 输入重定向文件，如 cmd < file */
    char *out_file;     /* 输出重定向文件，如 cmd > file */
    int   append;       /* 1 表示 >>，0 表示 > */
    int   background;   /* 1 表示后台 & */
} Command;
```

### 5.2 管道（命令链）

```c
#define MAX_PIPELINE 16

typedef struct {
    Command cmds[MAX_PIPELINE];
    int     count;      /* 管道段数量，至少为 1 */
} Pipeline;
```

### 5.3 别名

```c
typedef struct AliasNode {
    char *name;                 /* 别名，如 "ll" */
    char *value;                /* 替换串，如 "ls -l" */
    struct AliasNode *next;
} AliasNode;
```

### 5.4 历史记录

```c
#define HISTSIZE 1000

typedef struct {
    char *entries[HISTSIZE];
    int   count;        /* 当前条目数 */
    int   start;        /* 环形缓冲区起始（可选） */
} History;
```

## 6. 模块详细设计

### 6.1 输入模块（input.c）

**职责：** 使用 GNU readline 读取用户输入，提供行编辑与 Tab 补全入口。

**提示符格式：**

```
myshell:当前目录$
```

- 通过 `getcwd()` 获取当前目录
- 普通用户显示 `$`，若 `getuid() == 0` 显示 `#`（对齐实验文档）

**退出方式：**

- 内置命令 `exit`
- `readline` 返回 `NULL`（Ctrl+D）

### 6.2 解析模块（parser.c）

**职责：** 将原始字符串解析为 `Pipeline` 结构。

**解析规则：**

| 符号 | 处理 |
|------|------|
| 空格 / Tab | 参数分隔（引号内除外） |
| `"..."` / `'...'` | 保留为一个参数 |
| `\|` | 管道分段 |
| `<` | 输入重定向，作用于当前段 |
| `>` | 输出重定向（覆盖） |
| `>>` | 输出重定向（追加） |
| `&` | 后台标志，仅作用于整条 pipeline 末尾 |

**示例：**

```
grep hello a.txt | grep hello > out.txt &
```

解析为 2 段命令（均为内建）：

1. `grep hello a.txt`
2. `grep hello`，输出重定向到 `out.txt`，整条 pipeline 后台执行

**错误处理：**

- 空管道段、重定向缺少文件名 → 打印错误，不执行
- 管道段数超过 `MAX_PIPELINE` → 报错

### 6.3 内置命令模块

**内建命令列表（11 个）：**

| 命令 | 语法 | 行为 | 实现文件 |
|------|------|------|----------|
| `cd` | `cd [目录]` | 无参数 → `chdir($HOME)`；有参数 → `chdir(目录)` | `builtins.c` |
| `echo` | `echo [参数...]` | 输出参数，支持 `$VAR` | `builtins.c` |
| `pwd` | `pwd` | `getcwd()` 输出当前目录 | `builtins.c` |
| `ls` | `ls [-l] [path]` | 列目录；`-l` 详细模式 | `builtin_ls.c` |
| `cat` | `cat [file...]` | 读文件或 stdin 输出 | `builtin_cat.c` |
| `grep` | `grep [-i] pat [file...]` | 子串匹配过滤行 | `builtin_grep.c` |
| `type` | `type 命令名` | 内建 → "`命令` is a shell builtin"；否则 not found | `type.c` |
| `history` | `history [n]` | 列出历史；`n` 为最近 n 条 | `history.c` |
| `alias` | `alias` / `alias name=value` | 无参数列出全部；有参数创建别名 | `alias.c` |
| `unalias` | `unalias name` | 删除别名 | `alias.c` |
| `exit` | `exit` | 退出 Shell | `builtins.c` / `shell.c` |

**内建判定：**

- 维护 11 项内建命令名表（`type.c`），`type` 与 Tab 命令补全共用
- **不再**遍历 PATH 查找外部可执行文件

**为何 `cd` 必须是内置：**

`chdir()` 只影响调用进程。若 fork 子进程再 `cd`，仅子进程目录改变，Shell 自身目录不变，不符合预期。

### 6.4 执行模块（executor.c）

#### 6.4.1 单条内建命令

```
单内建、无重定向、非后台 → 父进程直接 run_builtin()
单内建、有重定向、非后台 → run_builtin_in_parent()（save/dup2/restore，不 fork）
单内建、后台或有特殊需求   → fork 子进程 → run_builtin() → wait 或 SIGCHLD 回收
非内建命令                 → 报错 "command not found"，返回 127
```

#### 6.4.2 管道执行

对 n 段命令，创建 n-1 个 pipe，n 个子进程，**每段必须是内建**：

```
cmd1 | cmd2 | cmd3

pipe1: cmd1 stdout → pipe1 → cmd2 stdin
pipe2: cmd2 stdout → pipe2 → cmd3 stdin
```

每个子进程关闭不用的 fd，设置 `dup2` 后调用 `run_builtin()`（不再 execvp）。

#### 6.4.3 重定向

在子进程或 `run_builtin_in_parent` 中设置：

| 语法 | 实现 |
|------|------|
| `cmd > file` | `open(file, O_WRONLY\|O_CREAT\|O_TRUNC, 0644)` + `dup2(fd, STDOUT_FILENO)` |
| `cmd >> file` | `open(file, O_WRONLY\|O_CREAT\|O_APPEND, 0644)` + `dup2` |
| `cmd < file` | `open(file, O_RDONLY)` + `dup2(fd, STDIN_FILENO)` |

**作用域：** 重定向只作用于其所在的那一段命令（与 bash 行为一致）。

#### 6.4.4 后台运行

- 解析器识别行末 `&`
- 父进程 `fork` 后不调用 `waitpid`，打印 `[pid] running in background` 后回到读入循环
- 子进程在后台独立运行直至结束
- 子进程退出后由 `SIGCHLD` 处理函数通过 `waitpid(-1, NULL, WNOHANG)` 非阻塞回收

#### 6.4.5 子进程回收与信号处理

**问题：** 内建 + 重定向若不当 fork，可能与 SIGCHLD 处理函数竞态，出现 `waitpid: No child processes`。

**本方案实现（`executor.c`）：**

| 场景 | 处理方式 |
|------|----------|
| 前台内建 + 重定向 | `run_builtin_in_parent()`，父进程内 dup2，**不 fork** |
| 后台子进程退出 | `SIGCHLD` 处理函数 `waitpid(-1, NULL, WNOHANG)` 回收 |
| 前台 fork wait | `wait_pid_retry()` 期间屏蔽 SIGCHLD |
| 管道多子进程 wait | `wait_pids()` 期间屏蔽 SIGCHLD |
| 兜底 | `wait_pid_blocking` 对 `errno == ECHILD` 视为已回收 |

### 6.5 历史模块（history.c）

**行为（对齐实验文档）：**

- 每条有效命令执行前写入历史
- 默认最多保存 `HISTSIZE`（1000）条，超出则丢弃最旧
- `history` 命令按序号打印
- `history n` 打印最近 n 条

**可选持久化：**

- 退出时写入 `~/.myshell_history`
- 启动时加载（便于演示，不影响核心功能）

**与 readline 的关系：**

- readline 自带上下键浏览；`history` 命令展示同一份历史数据

### 6.6 别名模块（alias.c）

**行为（对齐实验文档）：**

| 操作 | 语法 | 示例 |
|------|------|------|
| 创建 | `alias 别名='命令'` | `alias dir='ls -l'` |
| 查看 | `alias` | 列出所有别名 |
| 取消 | `unalias 别名` | `unalias dir` |

**展开时机：** 解析前对整行做别名替换（仅替换行首命令名，避免误替换参数）。

**展开方式：** 简单字符串替换，将 `dir arg1` 展开为 `ls -l arg1`，再交给 parser。

### 6.7 命令补全模块（completion.c）

**方案：** GNU readline + 自定义 `rl_attempted_completion_function`。

**补全范围：**

| 场景 | 策略 |
|------|------|
| 行首 / `\|` 后第一个 token | 仅补全 11 个内建命令名 |
| 路径参数 | readline 内置 `rl_filename_completion_function` |
| 双 Tab | readline 默认：唯一匹配补全，多个匹配列出候选 |

**示例（对齐实验文档）：**

- `more /root/teste` + Tab → `more /root/testexecvp.c`
- 连续双 Tab → 列出所有同前缀文件

## 7. 内建命令策略（不 exec 外部程序）

| 命令 | 实现方式 | 核心系统调用 |
|------|----------|--------------|
| `cd` | 内建 | `chdir` |
| `echo` | 内建 | `getenv`, `write`(stdio) |
| `pwd` | 内建 | `getcwd` |
| `ls` | 内建 | `opendir`, `readdir`, `stat` |
| `cat` | 内建 | `open`, `read`, `write` |
| `grep` | 内建 | `fopen`/`getline`, 字符串匹配 |
| `type` | 内建 | 查内建表 |
| `history` | 内建 | 内存 + `fopen` |
| `alias` / `unalias` | 内建 | 内存链表 |
| `exit` | 内建 | 退出标志 |
| 其他（如 `wc`） | **不支持** | 输出 `command not found` |

## 12. 风险与约束

| 项 | 说明 | 应对 |
|----|------|------|
| 仅 Windows 无 WSL | 无法运行 POSIX 系统调用 | 安装 WSL2 + Ubuntu |
| readline 未安装 | 编译失败 | `apt install libreadline-dev` |
| 复杂引号/转义 | 解析边界情况 | 第一阶段支持常见双/单引号，不做完整 bash 兼容 |
| 僵尸进程 | 后台子进程结束后未回收 | `SIGCHLD` 处理函数 + `waitpid(-1, WNOHANG)`；前台/管道 wait 期间屏蔽 `SIGCHLD`（**勿**用 `SIG_IGN` 与阻塞 waitpid 混用） |
| 非内建命令 | 用户输入 wc 等 | 输出 command not found，不 exec |

## 13. 小结

本方案以 **「读入 → 解析 → 内建执行（系统调用）→ fork 管道/后台」** 为主干，覆盖实验一全部必做功能：

- **命令：** ls、cd、cat、grep、echo、type  
- **Shell 特性：** history、alias、Tab 补全  
- **系统机制：** 后台 `&`、管道 `\|`、重定向 `>` / `>>` / `<`  

Tab 补全采用 **GNU readline（方案 A）**；开发在 Windows + Cursor，编译运行在 WSL2 Linux，源码可移植至其它 Linux 发行版。

确认本方案后，按第 9 节开发计划从 P1 开始编码实现。
