# myshell 课堂演示稿与演示思路

> 操作系统课程设计 · 实验一：Shell 编程  
> 适用对象：三人小组课堂演示（秦健超 / 张孙毓 / 郭沛特）  
> 预计总时长：**8～10 分钟**（含老师提问缓冲）

---

## 一、演示总思路

### 1.1 核心策略：先「能用」，再「懂原理」

老师最关心三件事：

1. **实验要求是否完成**（内建命令、管道、重定向、后台、历史、别名、Tab 补全）
2. **是否理解操作系统原理**（fork、pipe、dup2、系统调用 vs exec）
3. **是否自己写的**（能讲清模块分工和数据流转）

建议演示节奏：

```
开场（30s）→ 自动化测试背书（30s）→ 三人分工演示（各 2min）→ 总结（30s）→ 留时间答问
```

### 1.2 演示前准备（务必提前做）

在 WSL/Ubuntu 项目根目录执行：

```bash
cd /path/to/shell
make clean && make
make test          # 确认末尾 0 failed
./myshell          # 先跑一遍，确认能启动
```

**演示环境建议：**

- 终端字号调大（老师后排能看清）
- 提前在 `/tmp` 建好演示目录，避免现场 `cd` 出错
- 准备好 [项目讲解文档.md](./项目讲解文档.md) 的 **1.4 分层图** 和 **1.5 数据流图**（可投屏或打印）

**开场前可清理：**

```bash
rm -f /tmp/demo_out.txt /tmp/mylist.txt /tmp/demo.txt
```

### 1.3 项目一句话定位

> myshell 是用 C 语言实现的简易 Linux Shell。所有实验要求命令均在进程内用**系统调用**实现，**不通过 exec 调用** `/bin/ls` 等外部程序。

## 二、完整演讲稿（按顺序）

---

### 【第 0 段】开场 — 任意一人或秦健超（约 1 分钟）

**演讲稿：**

> 老师好，我们小组完成的是操作系统课程设计 **实验一：Shell 编程**，项目名称 **myshell**。
>
> Shell 是用户和 Linux 内核之间的命令解释器。用户输入一行命令，Shell 负责**读入、解析、执行**。我们的实现全部用 C 语言编写，核心特点是：
>
> - **允许系统调用**（`fork`、`pipe`、`dup2`、`open`、`opendir` 等）
> - **禁止 exec 外部程序**（不调用 `/bin/ls` 等系统命令）
>
> 也就是说，`ls`、`cat`、`grep` 都是我们自己在 Shell 里用系统调用实现的。
>
> 项目分三层：**读入与解析 → 调度 → 执行**。一条命令的数据流是：原始输入 → 别名展开 → 解析成 `Pipeline` 结构 → 记录历史 → 分发执行。
>
> 接下来我们先跑自动化测试，再分模块演示。

**同步操作：**

```bash
make test
```

若输出 `0 failed`，补一句：

> 「自动化测试覆盖了 15 个场景、22 条断言，包括管道、重定向、别名、后台和 Tab 补全。」

然后启动：

```bash
./myshell
```

---

### 【第 1 段】秦健超 — 解析与主控（约 2 分钟）

**演讲稿：**

> 我负责**命令读入、解析和主控流程**。
>
> 用户输入一行后，`main.c` 里的 `process_command()` 依次做：别名展开 → `parse_line()` 解析 → 记历史 → `shell_execute()` 分发。
>
> 解析器会把一行字符串拆成 `Pipeline` 结构，包含：命令参数、`|` 管道段、`<` `>` `>>` 重定向、行末 `&` 后台标志。
>
> 下面演示解析和分发。

**演示命令（按顺序敲）：**

```bash
pwd
ls -l | grep .c
type ls
type wc
```

**边敲边讲：**

1. `pwd` — 提示符显示当前目录，这是 `input.c` 用 `getcwd` 实现的。
2. `ls -l | grep .c` — 这条会被 `parse_line` 拆成两段：`ls -l` 和 `grep .c`，`count=2`，走管道执行路径。
3. `type ls` — 输出 `ls is a shell builtin`，说明是我们内建实现，不是外部程序。
4. `type wc` — 输出 `wc: not found`，我们**不 exec** 系统里的 `/bin/wc`，这是设计约束。

**若老师问解析细节，可答：**

- 引号内空格不拆分
- `>>` 优先于 `>`
- `&` 只在行末生效

**负责模块备忘：** 解析、主控、读入、Tab 补全、type（`main` / `parser` / `shell` / `input` / `completion` / `type`）

---

### 【第 2 段】张孙毓 — 进程、I/O、历史与补全（约 2.5 分钟）

**演讲稿：**

> 我负责**进程控制和 I/O 机制**，包括重定向、管道、后台运行，以及 `ls`、`grep` 内建命令。
>
> 重定向的本质是 `dup2` 替换文件描述符：把 stdout 从终端换成文件。
> 管道用 `pipe()` 创建匿名管道，每段命令 `fork` 一个子进程，用 `dup2` 连接读写端。
> 后台命令 fork 后父进程不 wait，靠 `SIGCHLD` 信号处理函数回收僵尸进程。
>
> 另外我维护了自动化测试脚本 `make test`。

**演示命令 — 重定向：**

```bash
echo hello_myshell > /tmp/demo_out.txt
cat /tmp/demo_out.txt

echo world >> /tmp/demo_out.txt
cat /tmp/demo_out.txt
```

**边敲边讲：**

- `>` 覆盖写，`>>` 追加写，底层是 `open` + `dup2`

**演示命令 — 管道：**

```bash
grep hello /etc/passwd | grep root
```

**边敲边讲：**

- 管道 `grep | grep` 创建子进程，前一段输出作为后一段输入

**演示命令 — 后台：**

```bash
sleep 2 &
```

**边敲边讲：**

- 后台会打印 `[pid] running in background`
- 父进程立刻回到提示符，不用等 sleep 结束
- 子进程结束后由 SIGCHLD handler 回收

**演示命令 — 历史与 Tab 补全：**

```bash
history 5
```

然后演示 Tab 补全（**重点，老师可能关注**）：

```bash
his<Tab>
```

按一次 Tab 应补全为 `history`；若多个匹配，按两次 Tab 列出候选。

**若老师问「重定向为什么不 fork」：**

> 前台单条内建 + 重定向时，在父进程里 `dup2` 执行再恢复，避免子进程快速退出后 SIGCHLD 和 wait 产生竞态。

**负责模块备忘：** 进程调度、I/O、内建 ls/grep、测试脚本（`executor` / `builtin_ls` / `builtin_grep`）

---

### 【第 3 段】郭沛特 — 内建命令、历史与别名（约 2.5 分钟）

**演讲稿：**

> 我负责**内建命令调度、history 和 alias**，以及 `cd`、`echo`、`pwd`、`cat`。
>
> 所有内建命令通过 `run_builtin()` 统一调度。`ls` 用 `opendir/readdir/stat`，`cat` 用 `open/read/write`，不调用系统自带程序。
>
> `history` 用环形缓冲区存储，持久化到 `~/.myshell_history`。
> `alias` 在解析前展开，但历史里存的是用户**原始输入**，和 bash 行为一致。

**演示命令 — 目录与环境变量：**

```bash
cd /tmp
pwd

echo $HOME
```

**演示命令 — 文件读取：**

```bash
echo test_line > /tmp/demo.txt
cat /tmp/demo.txt

ls -l
```

> 若需展示读系统文件，可用 `cat /etc/hostname`（输出短，适合现场）。

**演示命令 — 别名：**

```bash
alias ll='ls -l'
ll

alias
unalias ll

history 3
```

**边敲边讲：**

- `cd` 调用 `chdir` 改 Shell 进程工作目录，所以后面 `pwd` 变了
- `echo $HOME` 通过 `getenv` 读环境变量
- `alias ll='ls -l'` 之后输入 `ll`，实际解析的是 `ls -l`，但 history 里仍显示 `ll`

**亮点一句（收尾）：**

> 这就是我们「系统调用可以、系统命令不行」的设计——所有实验要求命令都在进程内自己实现。

**负责模块备忘：** 内建 cd/echo/pwd/cat、history、alias（`builtins` / `builtin_cat` / `history` / `alias`）

## 四、老师高频提问 & 标准答法

| 问题 | 怎么答 |
|------|--------|
| 为什么不用 `execvp` 调系统 ls？ | 实验目的是理解 Shell 和系统调用；exec 只是套壳调现成程序，我们用 `opendir/readdir/stat` 自己实现 |
| 管道怎么实现的？ | `pipe()` 建管道，每段 `fork` 子进程，`dup2` 把 stdout 接写端、stdin 接读端 |
| 重定向怎么实现的？ | `open` 打开文件，`dup2(fd, 1)` 把 stdout 指向文件 |
| 后台进程怎么回收？ | `SIGCHLD` 处理函数里 `waitpid(-1, NULL, WNOHANG)` 非阻塞回收 |
| 历史存在哪？ | 内存环形缓冲 + `~/.myshell_history` 文件持久化 |
| 别名什么时候展开？ | 解析前 `alias_expand_line()`，历史存原始行 |
| `wc` 为什么不行？ | 非内建命令，设计约束是不 exec 外部程序 |
| 你们怎么分工？ | 按模块三等分，见 [分工文档.md](../分工文档.md) 和 [项目讲解文档.md](./项目讲解文档.md) 1.4.3 |
| 怎么保证没 bug？ | `make test` 自动化测试 + 手动用例 |

## 五、应急预案

| 情况 | 处理 |
|------|------|
| `make test` 有失败 | 演示前务必重跑；若现场失败，改用手动命令演示，事后说明环境问题 |
| Tab 补全不灵 | 说明「需按一次/两次 Tab」，改演示 `type ls` 等不依赖补全的命令 |
| `cat /etc/passwd` 输出太长 | 改用 `/etc/hostname` 或 `/tmp/demo.txt` |
| 后台 sleep 看不出效果 | 改 `sleep 5 &`，边等边继续敲 `pwd` 证明不用等 |
| 老师追问源码 | 打开 `main.c`（流程）、`parser.c`（解析）、`executor.c`（fork/pipe） |
